## 4. LangGraph 워크플로우

```
planner → financial → news → bio_domain → disclosure
       → risk_scoring → supervisory → loan_decision → report_writer → END
```

각 노드는 `BioAgentState(dict)` 를 입력받아 해당 필드를 채워 반환합니다.

---

## 5. OpenAI API 연동 상태

- **SDK:** `openai` 패키지 사용
- **모델:** `gpt-5.4-mini`
- **API 키:** `.env`에서 `OPENAI_API_KEY` 로드
- **Fallback:** API 실패 시 로컬 규칙 기반 보고서 자동 생성 (앱 정상 동작)
- **현재 상태:** API 키 quota 초과로 Fallback 동작 중

## 6. 점수 산출 공식 검증

```
최종점수 = 재무(40%) + 뉴스(25%) + 바이오(25%) + 공시(10%)
         뉴스 없음 → 중립 50점 + missing_news=True 플래그
         공시 LOW=100 / MEDIUM=60 / HIGH=20
```

| 등급 | 점수 범위 | 판단 |
|------|----------|------|
| A | ≥85 | 승인 |
| B | 70~85 | 승인 (특이사항 없을 때) |
| C | 55~70 | 전문가검토 |
| D | 40~55 | 부결 |
| E | <40 | 부결 (특이사항 시 전문가검토) |
